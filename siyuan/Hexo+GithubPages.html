Hexo+GithubPages

阿里云服务器续费太贵，买新的搬运数据太麻烦，还是选择使用 Github Pages + Hexo 来建立博客

GitHub+Hexo 搭建个人网站详细教程 - 知乎 (zhihu.com)

安装 Hexo

  • 下载 Node.js (nodejs.org) 并默认安装即可，CMD 执行 node -v 与 npm -v 判断是否安装成功

  • npm install -g hexo-cli 安装 Hexo

  • 选择目标位置初始化博客文件夹 hexo init Blog

  • CMD 进入 Blog 文件夹后执行 hexo new test_my_site,hexo g,hexo s 完成博客建立，浏览器打开 localhost:4000 即可查看效果

配置并关联 Github

  • 创建 ssh 密钥关联 github

  • 创建个人仓库 用户名.github.io

  • Blog 根目录配置 _config.yml

deploy:
  type: git
  repo: https://github.com/用户名/用户名.github.io.git
  branch: master

  • CMD 执行 npm install hexo-deployer-git --save,hexo clean,hexo g,hexo d

  • 此时打开网页 用户名.github.io 即可查看效果

关联域名

  • 阿里云购买域名

  • 解析域名如图设置

其中_dbsauth 是设置 https 使用的，不用手动配置

  • 打开 Github 仓库设置中 Pages 项的 Custom Domain 填写购买的域名 (此处虽然会提示出错，但完成步骤后开始可以使用购买的域名直接访问博客)

  • 进入 Blog/source 文件夹建立记事本也填写购买的域名并保存为 CNAME，注意没有 txt 后缀

  • CMD 执行 hexo clean,hexo g,hexo d

配置 Https

阿里云域名配置 https（免费版）_a873217486 的博客-CSDN 博客

  • 申请完 SSL 证书后不用下载，阿里云会提示通过域名解析进行配置

  • 再回到 Github 设置页面，Custom Domain 不提示报错，最后打开下面的 Enforce HTTPS 选项即可使我们的域名支持 HTTPS 访问
